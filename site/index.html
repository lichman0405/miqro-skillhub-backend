Frontend Integration

Miqro-SkillHub is a self-hosted agent skill registry backend. This site is the frontend developer documentation — it explains how a browser SPA (Vue, React, etc.) connects to the backend via the TypeScript SDK and the frontend read-model API.

Start here: Import SkillHubClient from @miqro/skillhub-client, call client.frontendSearch(...), and render availableActions. Everything else follows from these three.

Core principles

  • Use SkillHubClient — the TypeScript SDK handles auth, URL encoding, error normalization, and pagination. Do not start with raw fetch.
  • Use availableActions — never derive permissions from role strings on the frontend. Every read model includes boolean action flags. Render buttons only when the matching flag is true.
  • Refetch the read model after every mutation — a successful mutation (approve, reject, create, update, delete) makes the previously fetched read model stale. Always refetch so availableActions and data stay current.
  • Do not infer permissions from role strings — authorization is enforced server-side. The frontend renders UI hints; the backend denies unauthorized mutations with 403.
  • Do not prefetch unbounded queues — review and promotion queues cap at 100 items per page. Use explicit page / size for user-driven paging. SDK iterators are for bounded background prefetch only — always set maxPages.

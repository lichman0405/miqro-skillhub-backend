Local development setup

The backend is a single Go binary. For frontend development you need the backend running with CORS enabled for your frontend origin.

Minimal backend (local storage mode)

Suitable for a single frontend developer on one machine.

# PowerShell (Windows)
$env:SKILLHUB_DATABASE_URL = "postgres://skillhub:skillhub@localhost:5432/skillhub?sslmode=disable"
$env:SKILLHUB_CORS_ALLOWED_ORIGINS = "http://localhost:5173"
$env:SKILLHUB_STORAGE_PROVIDER = "local"
$env:SKILLHUB_STORAGE_ROOT = "./data/storage"
$env:SKILLHUB_LOCAL_MODE = "true"
cd server
go run ./cmd/skillhub-migrate
go run ./cmd/skillhub-server
# Bash (Linux/macOS)
export SKILLHUB_DATABASE_URL="postgres://skillhub:skillhub@localhost:5432/skillhub?sslmode=disable"
export SKILLHUB_CORS_ALLOWED_ORIGINS="http://localhost:5173"
export SKILLHUB_STORAGE_PROVIDER="local"
export SKILLHUB_STORAGE_ROOT="./data/storage"
export SKILLHUB_LOCAL_MODE=true
cd server
go run ./cmd/skillhub-migrate
go run ./cmd/skillhub-server

Full Docker Compose stack

Runs PostgreSQL, Redis, MinIO, and the server with all integrations wired.

# Start infrastructure services
docker compose up -d postgres redis minio

# Start the full development profile (server + worker + minio-init)
docker compose --profile full up -d
The --profile full Compose uses SKILLHUB_LOCAL_MODE=true and minioadmin credentials — it is a local development profile, not a production configuration.

Windows notes

Docker Desktop and make may not be available on Windows. Direct Go commands are the baseline verification path: 
# These work on any platform with Go installed
cd server
go test ./...
go vet ./...
go build ./cmd/skillhub-server
go build ./cmd/skillhub-migrate
go build ./cmd/skillhub-worker

CORS configuration

The backend must explicitly allow the frontend origin. Cookie-based auth requires an explicit origin, not a wildcard.

Env varExampleNotes
SKILLHUB_CORS_ALLOWED_ORIGINS http://localhost:5173 Comma-separated origins. No wildcard for credentialed requests.

Redis / S3 / MinIO: dev vs production

ComponentDev (local)Production
Object storageLocal filesystem (SKILLHUB_STORAGE_PROVIDER=local)S3 / MinIO (SKILLHUB_STORAGE_PROVIDER=s3)
SessionsIn-memory (local mode)Redis (SKILLHUB_SESSION_BACKEND=redis)
Rate limitingIn-memory (SKILLHUB_RATE_LIMIT_BACKEND=memory)Redis (SKILLHUB_RATE_LIMIT_BACKEND=redis)
Production must use: S3/MinIO object storage, Redis-backed sessions, and Redis-backed rate limiting. Local filesystem storage is unsafe for multi-instance deployments. The production mode rejects local storage unless SKILLHUB_ALLOW_LOCAL_STORAGE_IN_PRODUCTION=true is set as an explicit emergency override.

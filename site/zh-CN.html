Miqro-SkillHub 前端对接 — 中文快速入口

一句话结论： 后端能力足够前端开工。当前缺的不是后端接口，而是 UI 工程。你可以开始写前端页面了。

英文文档链接（细节以英文为准）

Frontend API 页面到 API 的完整对照表 TypeScript SDK SkillHubClient 初始化、auth、unwrap、分页、错误处理 Local Dev 后端本地运行、CORS、Docker、Windows 注意事项 Fixtures Contract 响应示例，可用于 mock / Storybook

推荐技术栈

  • Vue 3
  • Vite
  • TypeScript
  • Vue Router
  • Pinia 或 TanStack Query / Vue Query

第一个可用页面应该是 skill discovery / search，不是营销首页。

第一周开发顺序

  1. App shell
    Header、navigation、search box、error boundary、SDK provider。
  2. Auth
    Login 页。用 client.login(username, password) 和 client.me()。
    401 跳 /login，403 显示无权限。
    logout 暂时 raw fetch POST /api/v1/auth/logout。
  3. Search
    / 和 /search。
    调用 client.frontendSearch(...)。filters、sort、pagination。
  4. Skill detail
    /skills/:namespace/:slug
    /skills/:namespace/:slug/versions/:version
    使用 frontend read-model，不要自己拼底层接口。
  5. Namespace
    /namespaces、/namespaces/:namespace
  6. Release + CI
    release list/detail、CI pipeline runs、checks / artifacts / gates
  7. Community
    issues、discussions、wiki、proposals
  8. Review / Promotion / Governance / Admin
    review queue/detail、promotion queue/detail、governance workbench、admin dashboard

核心规则

1. 优先用 SkillHubClient

import { SkillHubClient } from "@miqro/skillhub-client";

export const skillhub = new SkillHubClient({
  baseUrl: import.meta.env.VITE_SKILLHUB_API_URL ?? "http://localhost:8080",
  credentials: "include",
});

不要一开始就手写大量 fetch。

2. 权限按钮看 availableActions

const { data } = await skillhub.frontendSkillDetail(ns, slug);
if (data.availableActions.canApprove) {
  showApproveButton();
}

不要在前端自己推导 RBAC。 权限由后端返回的 boolean flags 决定。 即使前端显示了按钮，后端也会对未授权的 mutation 返回 403。

3. mutation 后必须 refetch

await skillhub.unwrap(skillhub.approveReview(reviewId, { comment: "OK" }));
// mutation 后重新拉取 read-model
const updated = await skillhub.unwrap(skillhub.frontendReviewDetail(reviewId));

4. 页面 loader 用 unwrap，表单错误用 envelope

// 页面 loader
try {
  const page = await skillhub.unwrap(skillhub.frontendSearch({ keyword: "agent" }));
} catch (err) {
  // error boundary 捕获
}

// 表单错误
const result = await skillhub.createIssue(ns, slug, input);
if (!result.success) {
  showError(result.error.message);
}

5. 不要无界预取

列表页使用 page / size 手动分页。SDK iterator 只用于有限后台预取，并且要设置 maxPages。

6. logout 临时方案

SDK 目前还没有公开的 logout() 方法。临时做法：

await fetch(`${apiBase}/api/v1/auth/logout`, {
  method: "POST",
  credentials: "include",
});

错误处理速查

HTTP 状态含义前端行为
401未认证或凭证过期跳转 /login
403已认证但无权限显示"你没有权限执行此操作"
404资源不存在显示"未找到"页面
409冲突（gate 失败等）显示后端返回的冲突原因
429请求过多显示限流提示 + 重试
503服务不可用显示"服务暂不可用"+ 重试

本地开发快速启动

SDK 引入（本地 file dependency）

// 前端项目的 package.json
{
  "dependencies": {
    "@miqro/skillhub-client": "file:../clients/typescript/skillhub"
  }
}

// 先构建 SDK
cd clients/typescript/skillhub
npm install
npm run build

后端启动（Windows PowerShell）

cd server
$env:SKILLHUB_DATABASE_URL = "postgres://skillhub:skillhub@localhost:5432/skillhub?sslmode=disable"
$env:SKILLHUB_CORS_ALLOWED_ORIGINS = "http://localhost:5173"
$env:SKILLHUB_STORAGE_PROVIDER = "local"
$env:SKILLHUB_STORAGE_ROOT = "./data/storage"
$env:SKILLHUB_LOCAL_MODE = "true"
go run ./cmd/skillhub-server

当前已知缺口

  • web/ 目录还没有正式前端工程
  • TS SDK 还没有发布到 npm，需要本地 file dependency
  • SDK 暂时没有 logout() 方法，先 raw fetch
  • CI step logs 暂未持久化
  • 前端页面设计系统还没确定

以上不是前端开工 blocker。

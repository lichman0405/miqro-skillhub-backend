Frontend read-model API

Every frontend page calls a dedicated read-model endpoint under /api/v1/frontend/. Each response includes the page data plus an availableActions object — boolean flags computed from the authenticated user's permissions. The frontend uses these flags to show or hide buttons, never role strings.

Rule: Read-model endpoints are the page contract. Mutations use portal routes (/api/v1/...). After any mutation, refetch the corresponding frontend read model so availableActions stays current.

CI routes (portal, no frontend read model)

Agent CI does not have a dedicated frontend read model. Use portal routes directly with the SDK's portal methods:

Review / promotion mutations

Review and promotion frontend endpoints are read-model only. Mutations (approve, reject, withdraw) use portal routes. After a mutation, refetch the frontend read model: 

// Approve a review
await client.unwrap(client.approveReview(reviewId, { comment: "LGTM" }));

// Refetch the read model so availableActions stays current
const updated = await client.unwrap(client.frontendReviewDetail(reviewId));

// Promotion mutations work the same way
await client.unwrap(client.approvePromotion(promotionId, { comment: "Promoting" }));
await client.unwrap(client.rejectPromotion(promotionId, { comment: "Not ready" }));
await client.unwrap(client.withdrawPromotion(promotionId));

Loading flow for every page

  1. Check auth — call client.me(). If 401, show login.
  2. Fetch read model — call the page's frontend SDK method.
  3. Render — use data for content, availableActions to show/hide controls.
  4. Mutations — call the corresponding SDK portal method, then re-fetch the read model.

Button visibility

Never hard-code role checks on the frontend. Use availableActions from the read model: 

const { data } = await client.frontendSkillDetail(ns, slug);
if (data.availableActions.canEdit) {
  // show edit button
}

Error states

StatusFrontend behavior
400Show validation error near the form/action
401Redirect to /login, preserving intended destination
403Show "You do not have permission"
404Show not-found page
409Show conflict reason, especially gate failures
429Show rate-limit message and retry affordance
503Show "Service unavailable" with retry

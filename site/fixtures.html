Contract fixtures

Committed example response envelopes for frontend read-model pages. Use them for mocks, Storybook stories, page skeletons, and UI demos.

Fixture location

server/internal/http/frontend/testdata/contracts/

Each file is a full { "success": true, "data": { ... } } envelope covering one of the ready frontend read-model routes.

These fixtures are example payloads, not a replacement for API tests. They represent current response shapes and are useful for UI development, but the backend is authoritative — fixture fields may drift from runtime responses over time.

How to use fixtures

Mock API (e.g., MSW)

import { http, HttpResponse } from "msw";
import searchFixture from "./contracts/search.json";

export const handlers = [
  http.get("/api/v1/frontend/search", () => {
    return HttpResponse.json(searchFixture);
  }),
];

Storybook stories

import searchFixture from "../../contracts/search.json";

export const Default = {
  args: {
    data: searchFixture.data,
  },
};

Page skeleton / layout work

// During early layout development, import fixtures to populate components
// before the backend is wired
const mockData = searchFixture.data;
render(<SearchPage searchResult={mockData.searchResult} />);

Empty / error state reference

Fixtures show the shape for populated data. Use the fixture structure to derive empty-state and error-state shapes:

  • Empty list: { tasks: [], pendingCount: 0, hasMore: false }
  • Error: { success: false, error: { code: "unauthorized", message: "..." } }
